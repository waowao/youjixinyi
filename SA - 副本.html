表单元素

搜索组件

搜索条件较少时，标签、搜索框、按钮置于同一行，顺序依次为标签（可选）-条件-文本框-按钮，各组件高度一致并居中对齐。标签视情况添加，一般建议缺省。

搜索条件较多时，采用合理的网格布局，每个搜索条件由标签和文本框/选择框构成，搜索按钮置于最下排，各个输入框大小应尽可能保持长宽一致。

标签和输入框不在一行的格式

编辑表单

  • 新增/编辑数据使用的表单，每行做多包括一个标签和输入框，各个输入框大小应尽可能保持长宽一致，上下对齐。
  • 如果用到checkbox或者radio，需使用Ace Admin的组件，因为浏览器原生的组件点击区域太小不方便使用。
  • 较短的输入提示，使用placeholder属性，也可以考虑使用js的tooltip插件。
  • 表单提交尽可能使用ajax的方法，避免全页面刷新。
  • 时间长的表单提交要有加载动画，以防止出现假死状态。需要避免用户重复点击的地方，点击后按钮变成disabled样式。
  • 表单提交前后有错误检测，出现输入错误的表单元素边框变成红色警告色，错误信息显示在输入框下方，同时页面聚焦到第一个有错误输入框。
  • 文本域的rows值合理，一般设为5行。

按钮

按钮布局

一般情况下，多种操作按钮并排与同一行，按钮较多、空间狭小（如表格操作列大于等于3时）时使用按钮组。

按钮按常用程度、语序排列。例如，新增>编辑>删除，上传>下载。

  • 刷新
  • 编辑
  • 屏蔽
  • 删除

按钮icon

为了区分按钮的功能，可以对按钮增加图标，图标置于文字之前。如果不确认使用什么icon建议不加，否则会显得很混乱。主要情况如下供参考。

按钮颜色

危险性的操作、不可恢复的操作按钮采用警告色红色

文档新增、编辑等影响数据库内容的操作采用绿色

点击后不会有任何影响的操作和表格最后一栏操作中的非按钮组按钮使用白色。

其他一般情况使用淡蓝色

页面布局

布局规则

  • 页面内容自上至下布局，顺序依次为：页面标题->页面介绍->(编辑表单|(搜索框|操作按钮)->表格数据)->备注。
  • 每行内容清晰、职责单一、尽可能不罗列过多内容。
  • 列表页跳转至详情页，详情页跳转至编辑页等情况，一般在本页跳转不新开页面。跳转后的页面应添加返回按钮，位于页头靠左。
  • 编辑操作，如新增、编辑、导出等按钮，位于分页数据上方，按钮居右并排显示。
  • 当前内容较多时，应对不同类的信息分模块。

提示（tip）

  • 所有提示单起一行与其他组件左对齐，一般使用p标签置于页头标题后。
  • 需要引起人十分注意的提示可以采用惊醒色、使用ace自带的alert等方法。
  • 需要弱化的可以加一个grey类，放在操作按钮、表单等的后面。

分页

  • 分页数据一般使用表格呈现，列标题居中，列内容居左。
  • 如果存在勾选行的批量操作，需使用Ace Admin的checkbox组件，复选框放在第一列，“勾选所有”位于最上方列表头单元格。
  • 页面数据尽可能使用ajax的方法加载。
  • 分页放置于页面内容的底部居右，每页显示数量应在20～200之间，一般每页为20条数据，可根据用户的实际使用场景增加单页显示数量。
  • 页码组件采用如下示例中的样式，具体使用方法请参考“编辑管理-作品管理-作品编辑”处的代码。未来会改成带跳页的组件，暂时还没有该组件。
  • 数据列非常多的情况，建议不用单纯的表格呈现，否则每列的数据都在狭窄的空间内，既丑又难用。
  • 状态值可考虑使用带色彩的小旗，色彩视情况而定，一般无效、删除、封禁等状态使用红色。
  • 表格列数较多的时候，设置合适的单元格和表格总宽度，同时增加一个横向滚动条。

提示条

一些提交后无需引起页面变化的用户操作，应有操作成功或失败的提示条，使用gui.info方法。

一些轻量级的错误，如搜索内容为空、批量操作未勾选条目等，也建议以提示条予以警示，使用gui.error方法。

提示条两秒后会自动消失，所以提示内容不宜过长或过复杂。 

gui.info('这是操作成功的提示！');

gui.error('这是出现报错的提示！');

注：每次操作5秒后才能再次操作。

警告框

一些内容较长或格式相对复杂的的通知消息、错误提示等，采取警告框的形式弹出提示用户，使用gui.alert方法。

该方法有两个参数，第一个参数是警告消息（可用html标签、必传），第二个参数是点击确认按钮后的回调。 

gui.alert('这是操作成功的提示！'，function(){
    console.log('这是个回调函数');
});

对话框

危险性操作需要用户二次确认，使用gui.confirm方法创建一个简单的双选对话框。

gui.confirm方法拥有3个参数，第一个参数是对话消息（可用html标签、必传），第二个参数是点击确认按钮后的回调，第三个参数是点击取消按钮后的回调。 

gui.confirm('确认删除所选的敏感词吗？', function() {
    gui.info('已成功删除4条记录！');
});

比较复杂的对话框，建议使用bootbox.dialog。 

$('#complex_dialog').click(function() {
    bootbox.dialog({
        title: '<h2 class="align-center">请先登录账号</h2>',
        message: '<div>' +
        '<label for="form-field-8">账号</label>' +
        '<input type="text" class="form-control" placeholder="邮箱/手机号">' +
        '<div class="space"></div>' +
        '<label for="form-field-8">密码</label>' +
        '<input type="password" class="form-control">' +
        '</div>',
        buttons: {
            cancel: {
                label: '取消',
                className: 'btn-white'
            },
            confirm: {
                label: "登录",
                className: "btn-primary btn-sm",
                callback: function() {
                    gui.info('登录成功！');
                }
            }
        }
    });
});

结语

本文档提供部分参考，开发中需根据实际情况进行调整。如需求方提出怪异的需求，建议以产品给出的设计为准。

表单的提交推荐使用ajax，这样做有很多好处：

  • 第一，避免全页面刷新，显得更流畅
  • 第二，使用固定格式返回结果，可以更清晰的查问题，而且容易处理错误
  • 第三，便于错误信息展示，可以用警示条、弹框等形式展示，对用户更友好

js代码中有很多冗余的代码，一些常用弹框、组件等可以采集下大家的意见，适当封装一下。

有些地方的代码有bug（如项目目录里gui.info组件样式错误），或插件、库类版本不兼容，建议做一下code review和优化。
